# Figma Output Format Guide

**Purpose**: Standardize design-analysis.md structure for consistent consumption by downstream agents.

**Required Sections**:
1. **Layout Structure** - Document component hierarchy with exact positioning (x, y coordinates), dimensions (width, height), spacing (margin, padding), and alignment rules using nested bullet lists or ASCII tree diagrams.
2. **Color Codes** - List all colors with HEX/RGB values, semantic names (e.g., `primary-500`, `text-muted`), and usage context (backgrounds, text, borders) in tabular format for easy reference.
3. **Image/Icon Positions** - Catalog all visual assets with file names, absolute/relative coordinates, dimensions, alt text, and parent component context to enable pixel-perfect implementation.
