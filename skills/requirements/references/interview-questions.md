# 확장 질문 은행 (Interview Questions Bank)

## 개요

이 문서는 Spec-Writer 에이전트가 각 Stage에서 사용자에게 물을 수 있는 질문을 카테고리별로 정리합니다. 모든 질문을 매번 묻는 것이 아니라, 프로젝트 컨텍스트에 맞는 질문을 선별하여 사용합니다.

---

## 1. 프로젝트 범위 (Stage 1: Project Understanding)

### 비즈니스 컨텍스트

| 질문 | 의미 | 영향 |
|------|------|------|
| "이 서비스의 주요 타겟 사용자는 누구인가요?" | 사용자 페르소나 결정 | UI 복잡도, 접근성 수준 |
| "MVP 출시 후 예상 사용자 규모는?" | 초기 스케일 결정 | 인프라 설계, DB 설계 |
| "수익 모델이 있나요? (무료/유료/프리미엄)" | 결제 통합 필요 여부 | API 설계, 보안 수준 |
| "경쟁 서비스나 참고하는 서비스가 있나요?" | 기능 범위 참고 | 요구사항 구체화 |
| "출시 목표 시기가 있나요?" | 범위 조정 | 기능 우선순위 |

## 8. 질문 선별 가이드라인

### 항상 물어야 하는 질문 (필수)
- 타겟 사용자
- MVP 기능 범위 확인
- 인증 전략
- 비기능 요구사항 (성능/보안)

### 컨텍스트에 따라 물어야 하는 질문 (조건부)
- 실시간 기능이 디자인에 암시된 경우 → 실시간 관련 질문
- 결제 UI가 있는 경우 → 결제/보안 질문
- 사용자 역할이 여러 개인 경우 → RBAC 질문
- 검색바가 있는 경우 → 검색 요구사항 질문
- 파일 업로드 UI가 있는 경우 → 업로드 질문

### 물어보지 말아야 하는 질문 (금지)
- 디자인에서 명확한 답이 있는 경우
- 이전 Stage에서 이미 답변된 경우
- 기술적으로 한 가지 선택만 합리적인 경우
